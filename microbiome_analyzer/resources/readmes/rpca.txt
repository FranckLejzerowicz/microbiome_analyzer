------------------------------------------
Robust Principal Component Analysis (RPCA)
------------------------------------------
RPCA is built for sparse microbiome compositional data with missing or uneven
sampling. It addresses shortcoming in standard PCA by treating the data matrix
as having a low-rank signal corrupted by sparse noise — a classic assumption
from the robust PCA / matrix completion literature.

RPCA decomposes a samples*features count matrix M in L + S components, with:
L = low-rank matrix capturing the true biological structure
S = sparse matrix capturing outliers/noise (e.g., contaminants, errors)

The analyses steps are:
1. Centered log-ratio transformation (Aitchison geometry): since CLR breaks with
 zeros (needs pseudocount), RPCA treats zeros as missing values rather than true
  absence.
2. Matrix completion over missing/zero entries: zeros are masked and the
low-rank structure is imputed during the optimization. Instead of imputing
zeros, the low-rank signal itself informs what those zeros "should be".
3. Robust to outliers: component S absorbs extreme values, so individual
anomalous samples or features don't distort the principal components.

In output, RPCA produces beta diversity and ordination files with sample and
feature loadings to identify which ASVs/taxa drive variation.
